Removing Refusal Directions from Transformer Weights

Find a hidden-state direction that separates harmful from harmless prompts at each layer, then stop the model from writing along it through its residual write-back matrices.

Gemma 4 E2B · 35 layers · dim 1536 · fp16 MPS

02

Capture Activations

Hook every layer, grab last-token hidden state.

Hℓ = harmful acts  Sℓ = harmless acts
shape [200, 35, 1536] on CPU
Intent (refuse/comply) encoded at last token position
03

Winsorize

Clamp outliers so extremes don't skew the mean.

q = quantile(|x|, 0.995)
x = clamp(x, −q, q)
Robust mean with 99.5% of information retained
04

Centroids & Difference

Average each class, difference = raw refusal direction.

μH = mean(H*)  μS = mean(S*)
Δℓ = μH,ℓ − μS,ℓ
Δ points from harmless toward harmful at each layer
05

Orthogonalize

Remove overlap with harmless — keep only refusal-specific signal.

ĥℓ = μS / ‖μS‖
r = Δ − (Δ · ĥ) ĥ  ×2 passes
r̂ℓ = r / ‖r‖  unit refusal dir
Clean unit vector per layer, orthogonal to helpful behavior
06

Project Out of Weights

Edit o_proj + down_proj to block the refusal direction.

Pℓ = I − r̂ r̂⊤  projector
W′ = PW  = W − r̂(r̂ᵀW)
Writes every direction except r̂ℓ
07

Restore Row Norms

Rescale each row to its original magnitude.

row′i = proji × ‖oldi‖ / ‖newi‖
Prevents degraded output from shrunk magnitudes
08

Repeat × 35 Layers

Each layer uses its own r̂ℓ.

35 layers × 2 matrices = 70 edits
✓ Refusals drop, coherence preserved
